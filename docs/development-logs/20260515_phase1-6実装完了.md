## Phase 2: 型 & データ層

### 実装内容
- `src/types/` 全型定義（player, point, session, court, analysis, index）
- `TennisSession` を discriminated union（`HardTennisSession | SoftTennisSession`）に設計
- `src/services/storage/`（StorageAdapter interface + AsyncStorageAdapter 実装）
- `src/stores/`（playerStore, sessionStore — Zustand v5 + persist）
- `src/data/mockSessions.ts`（3件のサンプルセッション）
- `src/data/coachingRules.ts`（7パターンのコーチングルール）

### 技術的決定
- Zustand の `createJSONStorage(() => defaultStorage)` で StorageAdapter 経由の疎結合を実現
  - ストア本体が `AsyncStorage` を直接 import しないよう設計（将来の SQLite / Supabase 差し替え対応）
- `TennisSession` を discriminated union にすることでソフトテニス固有フィールド（`position?: SoftTennisPosition`）を型安全に保持

### レビュー修正
- 初回: stores が `AsyncStorage` を直接 import → `defaultStorage` 経由に修正
- 初回: `TennisSession` が flat interface → discriminated union に変更

### 変更ファイル
`src/types/*`, `src/services/storage/*`, `src/stores/*`, `src/data/*`

## Phase 3: 分析サービス

### 実装内容
- `src/services/analysis/TennisAnalyzer.ts`（interface）
- `src/services/analysis/ManualAnalyzer.ts`（手動入力ベースの分析実装）
  - 1stサーブ成功率、2ndサーブ成功率、ダブルフォルト数、得点率、平均ラリー数
  - 7パターンの弱点判定ロジック
  - 強み（4条件）の文言生成
- `src/services/analysis/CoachingTipsGenerator.ts`（priority 順ソート）
- `src/services/analysis/PracticeMenuGenerator.ts`（durationMin 昇順ソート）
- `src/services/analysis/index.ts`（`getAnalyzer()` シングルトン）
- `__tests__/ManualAnalyzer.test.ts`（3テストケース）

### 技術的決定
- `ManualAnalyzer` を `index.ts` から再エクスポートしない（`getAnalyzer()` 経由のみを強制）
- `secondServeInRate` の分母を `secondServeIn + doubleFaults` とする（全サーブポイント数ではない）
- ソフトテニス固有の前衛コーチングは Phase 4 以降スコープとし、コード内コメントで明示

### レビュー修正
- `firstServeInRate` と `averageRallyCount` のテストに具体値 assertion を追加

### 変更ファイル
`src/services/analysis/*`, `__tests__/ManualAnalyzer.test.ts`, `jest.config.js`

## Phase 4: 共通UI / コートSVG / ポイント / 動画コンポーネント

### 実装内容
- `src/components/common/`（Button, Card, StatCard, SegmentedControl, Chip, SectionHeader, EmptyState）
- `src/components/court/`（CourtCanvas, CourtChart, CourtHeatmap）
  - SVG でテニスコートを描画（react-native-svg）
  - ソフトテニスはコートサーフェス色を `colors.navyLight` に変更して sport prop を可視化
- `src/components/point/`（PointLoggerForm, ServeResultPicker, ShotTypePicker, ResultReasonPicker, RallyCountStepper）
- `src/components/video/`（VideoPickerSheet, VideoRecorder, VideoThumbnail）
- `src/services/video/`（videoPicker.ts, videoRecorder.ts）

### 技術的決定
- すべてのコンポーネントは props 経由でデータを受け取り、store を直接購読しない
- コート色（courtSurface, courtLine, courtNet）を `colors.ts` に追加してハードコードを排除
- ショット色（courtBlue, courtPurple, courtCyan）も `colors.ts` に追加

### レビュー修正
- `CourtCanvas.tsx` のハードコード色3値を `colors.*` に移動
- `serveResults.ts` / `shotTypes.ts` の color フィールドを `colors.*` 参照に変換
- `Card.tsx` に `accessibilityLabel` prop を追加
- `CourtCanvas` が sport prop に応じてコートサーフェス色を切り替えるよう実装

### 変更ファイル

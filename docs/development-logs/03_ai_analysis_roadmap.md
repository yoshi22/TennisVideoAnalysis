# AI Analysis Roadmap

## v0: ManualAnalyzer

現在の分析は `ManualAnalyzer` による手動入力ベースです。

入力:

- セッション情報。
- ユーザーが記録した `PointRecord[]`。
- サーブ結果、ショット種別、結果理由、ラリー数、ショット位置。

出力:

- 1st サーブ成功率。
- 2nd サーブ成功率。
- ダブルフォルト数。
- エース数。
- 得点率。
- 平均ラリー数。
- 強み。
- 弱点パターン。
- コーチング Tips。
- 練習メニュー。

設計方針:

- MVP では動画解析に依存せず、ユーザーが入力した構造化データから確実に価値を出す。
- `TennisAnalyzer` interface を境界にして、将来の AI 分析実装と差し替えられるようにする。
- `WeaknessPattern` と `CoachingRule` を分離し、検出ロジックと表示コメントを独立して改善できるようにする。

## v1 ロードマップ

### Computer Vision によるボールトラッキング

動画からボール位置をフレームごとに推定し、ショット位置、球速、軌道の基礎データを取得します。

想定出力:

- ボール軌道。
- バウンド位置。
- インパクト推定位置。
- コート座標への正規化。

### ショット自動分類

映像とボール軌道から、サーブ、フォアハンド、バックハンド、ボレー、スマッシュ、ロブ、ドロップを自動分類します。

v0 の `ShotType` と互換性を保ち、手動修正できる UI を併用します。

### サーブ速度推定

サーブ開始、インパクト、バウンド、到達時間から速度を推定します。

MVP 後は厳密な計測値ではなく、練習内の相対比較と改善傾向を優先します。

### フォーム解析

姿勢推定を使い、以下を分析します。

- トス位置。
- 打点の高さ。
- 肩、肘、膝の角度。
- 体重移動。
- スイングタイミング。

### 自動ハイライト生成

ポイント結果、ラリー数、エース、ウィナー、長いラリー、失点パターンをもとに、見返すべき動画区間を抽出します。

### LLM によるパーソナライズドコーチングコメント

ManualAnalyzer や CV 分析の構造化結果を LLM に渡し、プレイヤーの競技、利き手、プレイスタイル、過去傾向に合わせたコメントを生成します。

想定入力:

- `PlayerProfile`
- 直近セッションの `TennisAnalysisResult`
- 過去セッションの傾向
- ユーザーが設定した練習目標

出力方針:

- 短い改善コメント。
- 次回練習メニュー。
- 試合前チェックリスト。
- 保護者やコーチ向けの共有サマリー。

## v2 ロードマップ

### ダブルスのポジショニング分析

ソフトテニスと硬式ダブルス向けに、ペアの位置関係、前衛/後衛、雁行陣、並行陣、ローテーションを分析します。

指標候補:

- 前衛決定率。
- ポーチ成功率。
- 前衛の触球率。
- 後衛ラリー継続率。
- センターカバー率。
- ペア間距離。

### コーチ向け生徒管理機能

複数プレイヤーのプロフィール、セッション、課題、練習履歴を管理できるコーチ向け機能を追加します。

想定機能:

- チーム/部活単位の選手一覧。
- 選手ごとの弱点推移。
- 練習メニューの割り当て。
- コメント履歴。
- 保護者共有用レポート。

### Supabase へのデータ移行

ローカルファーストの AsyncStorage から、Supabase ベースの同期構成へ移行します。

移行対象:

- `PlayerProfile`
- `TennisSession`
- `PointRecord`
- 動画メタデータ
- 分析結果キャッシュ

移行方針:

- 既存の `StorageAdapter` と store 構造を活かし、段階的に repository 層を追加する。
- ローカル保存はオフラインキャッシュとして残す。
- 認証、チーム共有、コーチ権限を Supabase 側で管理する。
